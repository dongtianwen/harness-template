# 执行日志目录

> AI 每次执行任务时，必须在此目录写入日志文件。
> 不写日志视为任务不完整。

## 文件命名规则
```
log-task-{三位编号}-{YYYY-MM-DD}.txt
```
示例：`log-task-001-2026-03-28.txt`

## 日志格式（含性能监控）
```
任务：task-{id}
开始时间：{YYYY-MM-DD HH:MM:SS}
结束时间：{YYYY-MM-DD HH:MM:SS}
执行耗时：{N} 分钟

执行步骤：
  1. [{HH:MM}] {步骤描述}
  2. [{HH:MM}] {步骤描述}

验证命令及输出：
  $ {命令}
  {实际输出}

性能指标：
  - 文件读取数：{N} 个
  - 命令执行数：{N} 次
  - 验证轮次：{N} 轮（含重试）

结论：通过 / 不通过
备注：{可选}
```

## 性能监控用途
- 识别哪类任务耗时异常（超过预期 2 倍则记录原因）
- 追踪验证重试次数（重试超过 3 次说明任务定义有问题）
- 作为优化任务粒度的依据

## 保留规则
- 日志文件永久保留，不允许删除
- 如需归档，移至 logs/archive/ 子目录
