# Skill：验证协议（verification）

## 触发条件
当任务描述包含以下关键词时自动加载：
- "验证"、"验收"、"测试"、"verification"、"evaluate"、"Evaluator"

## 用途
作为独立 Evaluator，从用户视角验证任务交付物，消除 AI 自我评估偏差。

## 工作流程
1. 读取对应 task 文件的验收标准
2. 不看代码实现过程，只看交付结果
3. 按以下优先级选择验证工具：
   - Chrome DevTools MCP 可用 → 优先使用浏览器工具
   - 长期迭代项目有 Playwright 脚本 → 运行 playwright test
   - 其他 → 运行命令行验收命令
4. 对每一项验收标准输出 Yes / No + 实际证据
5. 生成完整验收报告

## 输出格式
```
# 验收报告：Task-{id}

## 验证工具
{使用的验证工具}

## 验收结论
| 验收项 | 结果 | 证据 |
|--------|------|------|
| {项目} | Yes/No | {实际输出或截图描述} |

## 结论
通过 / 不通过

## 若不通过
- 原因：
- 建议修复方案：
```

## 注意
- 任何一项为 No，不允许标记任务为已完成
- 验证结果必须有真实证据，不接受描述性语言
