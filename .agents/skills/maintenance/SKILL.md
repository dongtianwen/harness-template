# Skill：定期维护（maintenance）

## 触发条件
当任务描述包含以下关键词时自动加载：
- "维护"、"maintenance"、"清理"、"文档更新"、"技术债"

## 用途
定期对项目进行系统性自检，保持文档与代码状态一致。

## 工作流程
1. 扫描 docs/ 下所有文件，标记与 src/ 不一致的描述
2. 检查 tasks/ 中任务状态是否准确
3. 扫描 src/ 中 TODO / FIXME 注释，更新 docs/technical-debt/README.md
4. 校准 STATE.md，确保反映实际进度
5. 将发现的问题以列表形式输出，等待人工决策

## 输出格式
```
# 维护报告 - {YYYY-MM-DD}

## 过时文档
- {文件}：{问题描述}

## 任务状态异常
- {task-id}：{问题描述}

## 新发现技术债
- {描述}（优先级：高/中/低）

## STATE.md 校准项
- {变更描述}
```

## 注意
- 不直接修改文件，只输出报告
- 发现问题后由人工决定是否处理
