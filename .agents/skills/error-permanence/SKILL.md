# Skill：错误永久化（error-permanence）

## 触发条件
当任务描述包含以下关键词时自动加载：
- "修复"、"fix"、"bug"、"报错"、"失败"、"不通过"、"永久化"

## 用途
每次出错修复后，把根本原因转化成永久规则，让同类错误不再发生。

## 工作流程
1. 识别本次错误的根本原因（不是"代码写错了"，而是"为什么会写错"）
2. 判断规则归属：
   - 命名/结构/代码风格 → docs/conventions.md
   - AI 行为约束 → AGENTS.md
   - 环境/工具/部署 → docs/runbooks/README.md
   - 数据格式 → docs/data-schema/
   - 已知风险 → docs/risks/README.md
3. 把根本原因转化成一条可执行规则，追加到对应文件
4. 在任务报告中注明："已永久化为规则：{规则内容} → {文件路径}"

## 规则写法标准
```
# ✅ 好的规则（具体、可执行）
"引入图表库前必须在 requirements.txt 锁定精确版本，
 不允许使用 >= 或 ~ 版本范围"

# ❌ 坏的规则（模糊、无法执行）
"注意版本兼容性"
```

## 注意
- 每次修复必须产出至少一条规则
- 规则必须具体到可以直接执行或检查
- 同类错误出现第二次，说明上次永久化不到位，需要加强规则
