### 问题3：PowerShell 调用带空格路径的可执行文件失败
- 现象：命令中直接写入 `C:\Program Files\...` 这类路径时，被 PowerShell 拆成 `C:\Program`，导致“不是可识别的 cmdlet、函数、脚本文件或可执行程序”错误。
- 解决方法：在 PowerShell 中执行带空格路径的可执行文件时，必须使用调用运算符 `&`，或避免在管道末尾裸写可执行路径；优先改写为当前 shell 内可直接执行的多行脚本。
