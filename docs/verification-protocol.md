# AI 验证协议（verification-protocol.md）

> 本文件定义 AI 验证自身工作的标准流程。
> 任何任务的验收结论，必须基于本协议执行的真实结果，不接受推断或假设。

---

## 核心原则

**AI 必须自己运行验收命令，不能让人代为验证。**

"我认为应该能通过"不是验证。
"运行了命令，输出如下"才是验证。

---

## 标准验证流程（每个任务必须执行）

### Step 1：运行所有验收命令
- 逐条执行 task 文件中的验收标准命令
- 不允许跳过任何一条
- 不允许用"应该没问题"替代实际运行

### Step 2：记录真实输出
- 把命令和实际输出原文贴进报告
- 输出必须是真实终端输出，不是描述性语言
- 示例：
  ```
  # ✅ 正确写法
  命令：pytest tests/ -v
  输出：
    test_calculate_ucl PASSED
    test_detect_anomaly PASSED
    2 passed in 0.43s

  # ❌ 错误写法
  输出：（测试全部通过）
  ```

### Step 3：判断通过 / 不通过
- 输出与预期完全一致 → Yes
- 输出与预期有任何偏差 → No，必须说明原因

### Step 4：不通过时的处理规则
- 不允许把"No"标记为"Yes"
- 不允许在未修复的情况下关闭任务
- 必须在报告中写明：问题原因 + 修复方案 + 修复后重新验证结果

### Step 5：写进报告
- 验证结果章节必须包含：实际命令 + 实际输出 + Yes/No 结论
- STATE.md 必须在任务结束后更新

---

## 日志记录要求

每次任务执行，必须在 `logs/` 下写入执行日志：

**文件名格式：** `log-task-{id}-{YYYY-MM-DD}.txt`

**内容：**
```
任务：task-{id}
执行时间：{timestamp}
执行步骤：
  1. {步骤描述}
  2. {步骤描述}
验证命令及输出：
  {命令}
  {输出}
结论：通过 / 不通过
```

---

## 禁止行为

- 禁止在未实际运行命令的情况下填写验收结论
- 禁止把描述性语言当作命令输出
- 禁止在任何验收项为 No 时关闭任务
- 禁止跳过 STATE.md 更新

## 浏览器验证（界面类任务）

> 适用于任何涉及 Web 界面、UI 渲染、前端交互的任务。

### 优先使用浏览器工具验证
如果当前环境支持浏览器工具（如 Chrome DevTools、DOM 快照、页面截图、界面操作），**必须使用浏览器工具直接验证界面输出**，不能只依赖命令行输出。

能实现则实现，不能实现则降级为命令行验证，但必须在报告中注明"浏览器验证不可用，已降级为命令行验证"。

### 浏览器验证内容
- **DOM 快照**：确认关键元素存在（标题、图表、报警横幅等）
- **页面截图**：记录实际渲染结果，作为验收证据
- **交互验证**：模拟用户操作（点击、切换、输入），确认响应正确
- **控制台检查**：确认无 JS 报错、无 Warning

### 记录格式
```
浏览器验证：
  工具：Chrome DevTools / DOM 快照 / 截图
  验证项：{描述}
  结果：通过 / 不通过
  证据：{截图路径 或 DOM 片段}
```

### 禁止行为
- 禁止在界面类任务中跳过浏览器验证（若环境支持）
- 禁止用"页面看起来正常"替代实际浏览器工具输出

## 验证工具选择规则

> 根据项目类型选择验证工具，不强制要求所有项目都用同一套。

### 判断流程
```
有 Web 界面？
  ├── 是 → Chrome DevTools MCP 可用？
  │         ├── 是 → 优先使用 MCP（实时、无需写脚本）
  │         └── 否 → 是长期迭代项目？
  │                   ├── 是 → 使用 Playwright（可重复运行）
  │                   └── 否（Demo）→ 命令行验证 + 截图描述
  └── 否 → 命令行验证（pytest / curl / 终端输出）
```

### Chrome DevTools MCP 适用场景
- 一次性 Demo、原型验证
- 需要即时交互验证的场景
- 编辑器支持（Antigravity）

### Playwright 适用场景
- 长期迭代项目（需要回归测试）
- 每次改动后需要确保旧功能不被破坏
- 已有测试脚本的项目

### Generator 必须同时生成 Playwright 脚本的条件
满足以下全部条件时，Generator 必须在 `tests/` 下同时生成 Playwright 测试脚本：
- 项目有 Web 界面
- PRD.md 标注为"长期迭代项目"
- 当前任务包含 UI 交互功能

否则，Playwright 脚本为可选项。
